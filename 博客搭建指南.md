## 什么是博客网站

博客（Blog）是一个可以让你在互联网上发布文字、图片、视频等内容的个人网站。就像你的在线日记本，你可以：
- 分享你的想法和经验
- 记录你的学习过程
- 展示你的作品
- 与他人交流互动

## 需要准备什么

### 基础知识
- 会使用电脑和浏览器
- 基本的文件管理能力
- 不需要编程经验（我们会从头教你）

### 软件工具
- 文本编辑器（推荐 VS Code 或记事本）
- 网页浏览器（Chrome、Firefox 等）
- Git（用于版本管理，可选）

### 时间投入
- 初次搭建：2-3 小时
- 学习基础：1-2 周
- 持续改进：循序渐进

## 技术选择

对于新手，我推荐以下三种方式：

### 方式一：纯静态 HTML + CSS + JavaScript（最简单）
**优点：**
- 最基础，容易理解
- 不需要服务器
- 可以免费托管在 GitHub Pages
- 完全掌控你的网站

**缺点：**
- 需要手动管理内容
- 添加新文章要写 HTML

**适合人群：** 想从基础学起的新手

### 方式二：静态网站生成器（推荐）
**常用工具：**
- **Hexo**（中文文档完善，简单易用）
- **Hugo**（速度快，功能强大）
- **Jekyll**（GitHub 原生支持）

**优点：**
- 用 Markdown 写文章（像写 Word 一样简单）
- 自动生成网站
- 大量免费主题
- 免费部署

**缺点：**
- 需要安装 Node.js 或其他环境

**适合人群：** 想快速上手的新手（推荐！）

### 方式三：动态博客系统
**常用工具：**
- WordPress（最流行）
- Typecho（轻量级，中文支持好）

**优点：**
- 功能完整
- 有管理后台
- 插件丰富

**缺点：**
- 需要购买服务器
- 有一定学习成本
- 需要维护

**适合人群：** 有一定基础，想要更多功能

## 发布内容

### 写什么内容？

刚开始写博客，很多人会不知道写什么。这里有一些建议：

1. **自我介绍** - 第一篇文章可以介绍你自己
2. **学习笔记** - 记录你学习的知识
3. **生活感悟** - 分享你的想法和经历
4. **技术教程** - 教别人你学会的东西
5. **项目记录** - 展示你做的项目

### Markdown 基础语法

如果你用 Hexo 等工具，需要学习 Markdown。这里是基础语法：

```markdown
# 一级标题
## 二级标题
### 三级标题

**粗体文字**
*斜体文字*

- 列表项 1
- 列表项 2

[链接文字](https://网址)

![图片描述](图片地址)
```

## 部署上线

部署就是把你的博客放到互联网上，让别人可以访问。

### 免费部署方案

1. **GitHub Pages**（推荐）
   - 完全免费
   - 稳定可靠
   - 域名：用户名.github.io

2. **Netlify**
   - 免费
   - 部署简单
   - 支持自定义域名

3. **Vercel**
   - 免费
   - 速度快
   - 适合静态网站

详细步骤请查看 [部署指南](./教程/部署指南.md)

## 常见问题

### 1. 我完全不懂编程，能学会吗？
**答：** 当然可以！本教程就是为零基础的人准备的。只要跟着步骤走，一定能学会。

### 2. 需要花钱吗？
**答：** 不需要！使用静态博客 + GitHub Pages 完全免费。

### 3. 需要购买域名吗？
**答：** 不需要。GitHub Pages 会给你一个免费域名。如果你想要自己的域名（比如 www.你的名字.com），可以购买，但不是必需的。

### 4. 博客搭建好后怎么更新内容？
**答：** 
- HTML 方式：编辑 HTML 文件，上传到 GitHub
- Hexo 方式：写 Markdown 文件，运行命令生成和部署

### 5. 我的博客别人能找到吗？
**答：** 需要做 SEO（搜索引擎优化）。主要方法：
- 写高质量的原创内容
- 使用合适的标题和关键词
- 在社交媒体分享你的文章
- 向搜索引擎提交你的网站

### 6. 可以添加评论功能吗？
**答：** 可以！静态博客可以用第三方评论系统：
- Gitalk（使用 GitHub 账号）
- Disqus（国外流行）
- Valine（简单易用）

### 7. 遇到问题怎么办？
**答：** 
- 仔细阅读错误提示
- 搜索错误信息
- 查看官方文档
- 在技术社区提问（如 SegmentFault、V2EX）
